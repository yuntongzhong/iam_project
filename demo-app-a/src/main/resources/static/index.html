概念词典

把容易被混用的几个词拆开，便于现场说明“为什么 Claims 里有角色和权限，而代码里判断的是 Authority”。

角色 Role

角色是业务身份包，例如系统管理员、审计员。它通常代表一个岗位或职责集合。

权限 Permission

权限是最小动作单元，例如用户查询、应用写入、访问某个客户端。它回答“能做什么”。

Spring Authorities

Authority 是 Spring Security 运行期真正参与判权的字符串，例如 ROLE_ADMIN、SCOPE_openid。

Claims

Claims 是 IAM 写入 OIDC 身份上下文的数据集合，客户端可用它渲染页面，也可进一步映射为本地授权信息。
