## Design Context

### Users
这个项目主要面向两类人：
- 课程答辩老师，需要在短时间内看清楚 IAM、RBAC、SSO、MFA 是否真的跑通，以及系统是否具备工程化表达。
- 企业后台管理员或开发同学，需要快速理解用户、部门、角色、权限、应用客户端之间的关系，并能现场操作演示。

页面的核心任务不是做复杂日常运营，而是用清晰、可信、带技术含义的方式，把身份治理模型和 SSO 结果讲明白。

### Brand Personality
专业、可信、技术感。

界面应该像“可现场讲解的治理仪表台”，而不是通用后台模板。它需要让人感到结构严谨、信息有层次、适合答辩展示，同时避免花哨和廉价的炫技效果。

### Aesthetic Direction
采用亮色主题，整体偏“编辑化的企业控制台”方向：温润浅底、深墨文字、青绿色与赤陶色点缀，带少量技术图纸式装饰。布局强调信息编排和节奏感，让用户一眼看到身份治理全貌。

参考方向：
- 企业治理看板的可信度
- 产品发布页级别的排版精度
- 适合课堂投屏和桌面演示的高对比可读性

明确避免：
- Bootstrap 默认后台模板感
- 大面积廉价渐变和发光效果
- 纯功能堆叠、缺少叙事层次的 CRUD 页面

### Design Principles
1. 先讲清概念，再承载操作：角色、权限、Authority、Claims 必须有中文语义说明。
2. 优先服务答辩演示：关键关系、数据规模和安全能力要在首屏可见。
3. 保持可信与克制：视觉上有识别度，但不靠夸张效果取胜。
4. 同一视觉语言贯穿管理台与 Demo 应用，体现“同一 IAM 体系下的前后端一致性”。
5. 移动端不隐藏核心信息，只重排结构和交互密度。
