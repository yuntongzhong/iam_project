# 答辩演示脚本

## 演示顺序

1. 展示 `admin.html`，说明用户、部门、角色、应用、日志都在同一控制台管理。
2. 展示种子数据：
   - `admin`
   - 部门树：总部 / 研发中心 / 销售中心
   - 客户端：`app-a`、`app-b`
3. 下载用户导入模板，上传一个准备好的 UTF-8 CSV，先做预检，再执行正式导入。
4. 在控制台创建或修改一个用户，触发审计日志。
5. 展示应用客户端注册区，说明 `PKCE`、授权码 TTL、Access Token TTL、Refresh Token TTL 与 Refresh Token 轮换策略。
6. 访问 Demo App A。
7. IAM 登录输入账号密码，首次进入 TOTP 绑定页，完成扫码与验证码输入。
8. 登录成功后返回 App A，展示 Claims 与 `expiresAt`。
9. 不退出浏览器，直接访问 App B，展示无需重新输入密码即可完成 SSO。
10. 连续输入错误密码 5 次，展示账号被锁定。
11. 打开审计日志，展示：
   - 登录成功
   - 登录失败
   - 锁定事件
   - 管理操作
   - 敏感字段已脱敏
   - “目标系统”列能区分 IAM 控制台、App A、App B
12. 执行统一退出，再说明访问任一客户端都会重新进入认证流程。
